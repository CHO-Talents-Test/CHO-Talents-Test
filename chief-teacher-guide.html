📋 자주 하는 작업

  1. 관리 > 학생 일괄 등록에서 새 학기 또는 새 부서 학생을 엑셀로 등록합니다.
  2. 관리 > 사용자 관리에서 가입 신청과 부서 이동 요청을 확인합니다.
  3. 달란트 > 달란트 관리 상세 화면에서 잘못 지급된 달란트를 반환합니다.
  4. 상품 > 구매 통계에서 부서별 주문 흐름과 지연 상태를 확인합니다.
  5. 운영 > 보고서와 운영 > 버전에서 최근 변경 사항을 확인합니다.

💰 달란트 반환 (v3.48.0)

상세 팝업 요약 4칸

달란트 > 달란트 관리 > 상세 팝업 상단에는 다음 네 가지가 구분되어 표시됩니다.

  • 누적적립 — 지금까지 받은 달란트 합계
  • 총사용 — 실제 상품 구매 확정으로 차감된 달란트만 집계
  • 총반환 — 반환 처리된 달란트 (사용과 별도 집계)
  • 잔여 — 현재 사용 가능한 달란트

반환 처리 방법

  1. 상세 팝업에서 반환할 적립 내역을 찾습니다. 내역 유형은 적립, 사용, 반환 3종으로 구분됩니다.
  2. 반환 버튼을 누르고 사유를 입력한 뒤 확정합니다.
  3. 반환되면 해당 건은 총반환에 합산되고, 총사용에는 포함되지 않습니다.
  4. 같은 거래 ID 기준으로 1회만 반환할 수 있으며, 지급 모달의 취소 상태와 동기화됩니다.

🧭 권한 범위 이해

  • 부장 교사는 전체 현황을 볼 수 있지만, 사용자/관리자 수정은 담당 관리 부서와 권한 순위 조건을 따릅니다.
  • 구매 담당 교사가 아닌 부장 교사는 구매 처리에서 담당 관리 부서 범위가 적용될 수 있습니다.
  • 달란트 항목 관리의 퀵 버튼 설정은 가능하지만, 항목 CRUD와 QR 관리는 전도사님 이상 권한 기준을 따릅니다.
  • 로그/작업 이력 화면 직접 조회와 페이지 접근/기능 설정 변경은 관리자 권한입니다.
  • 권한명, 구매 상태, 상품 카테고리, 로그/작업 유형은 공통 코드 기준으로 표시됩니다. 명칭 변경은 개별 화면이 아니라 코드 마스터 기준을 확인합니다.
