📝 작업 이력 작성 룰

작업 이력(Audit Trail)에 기록되는 기준과 카테고리를 설명합니다. v3.51.0 기준 QR 유효 시간·위치 반경 변경도 QR 관리 작업 상세에 포함합니다.

📌 작업 이력 vs 로그의 차이

로그 (admin/logs.html): 모든 시스템 이벤트 기록 (페이지 방문, JS 에러 포함) — 레벨별 필터링
작업 이력 (admin/audit.html): 관리 작업만 필터링한 감사 기록 — 카테고리별 필터링
두 기능은 동일한 activity_logs 테이블을 사용합니다. 작업 이력은 기본 AUDIT_ACTIONS와 activity_logs.action 코드 마스터에 등록된 관리 액션을 표시합니다.
DB 권한 확인: 작업 이력이 비어 있고 로그도 쌓이지 않으면 docs/TASK-047_activity_logs_grants.sql을 적용해 activity_logs INSERT 권한과 RLS 정책을 복구합니다.

✅ 작업 이력에 포함되는 기준

기준설명
AUDIT_ACTIONS/코드 마스터 등록admin/audit.html의 기본 AUDIT_ACTIONS 또는 DB activity_logs.action 코드 그룹에 키가 등록된 액션이 작업 이력에 표시됩니다
관리 작업 중심사용자/부서/달란트/상품/주문 등의 CRUD 작업이 주 대상
시스템 자동 이벤트 제외PAGE_VIEW, JS_ERROR, PROMISE_REJECTION 등은 포함되지 않음
실패/거부도 일부 포함로그인 실패 등 보안 관련은 작업 이력에도 포함될 수 있음
