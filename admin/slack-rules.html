🔔 Slack 알림 룰

브라우저에서 발생한 운영 이벤트가 Supabase Edge Function을 거쳐 Slack 채널로 전송되는 기준입니다.

1. 전체 호출 구조

화면 이벤트 → sendSlackNotify(type, data) → _sb.functions.invoke('slack-notify') → Edge Function → Slack Incoming Webhook
구간담당 파일역할
브라우저 공통 유틸js/slack-notify.jssendSlackNotify(type, data)를 제공하고 동일 알림을 5초 동안 중복 전송하지 않습니다.
로그 알림js/activity-log.jsWARN, ERROR, FATAL, CRITICAL 로그 기록 시 log_alert를 Edge Function으로 보냅니다.
Edge Functiondocs/edge-function-slack-notify.ts알림 유형과 부서명에 따라 사용할 Slack Webhook Secret을 선택하고 Block Kit 메시지를 구성합니다.
Slack 호출은 fire-and-forget 방식입니다. 실패해도 구매, 가입, Q&A 같은 사용자 작업은 중단하지 않고 브라우저 콘솔에 경고만 남깁니다.

2. 알림 유형과 라우팅

구매 상태의 화면 표시명은 공통 코드북/DB product_orders.status 코드 마스터를 따릅니다. Slack 발송 조건은 표시명이 아니라 상태 코드 전환을 기준으로 유지합니다.

3. 필요한 Edge Function Secrets

Secret용도
SLACK_WEBHOOK_PART1 ~ SLACK_WEBHOOK_PART51부~5부 부서별 채널
SLACK_WEBHOOK_WORSHIP예배부 채널
SLACK_WEBHOOK_PRODUCT_MANAGEMENT상품 관리 채널
SLACK_WEBHOOK_OPERATIONS운영 로그 채널
SLACK_WEBHOOK_ANSWERQ&A 채널
Webhook URL 원문은 정적 HTML, JS, app_config에 저장하지 않습니다. Supabase Edge Function Secret 또는 서버 환경변수에만 저장합니다.

4. 신규 알림 추가 기준

  1. 이벤트가 발생하는 화면에서 sendSlackNotify('새_type', data)를 호출합니다.
  2. docs/edge-function-slack-notify.ts의 라우팅 함수와 메시지 생성 로직에 새 type을 추가합니다.
  3. 필요한 Webhook Secret을 Supabase Edge Function 환경변수에 추가합니다.
  4. 운영 문서와 이 페이지에 라우팅 기준, 포함 정보, 실패 시 동작을 기록합니다.
