📋 로그 작성 룰

시스템 로그가 기록되는 기준과 규칙을 설명합니다. v3.59.0 기준 인증/권한 리디렉트 원인, QR 위치 권한 차단, 상품 카테고리 추가, 공지 관리도 로그로 구분합니다.

📊 로그 레벨 체계

7단계 로그 레벨을 사용하며, ERROR 이상은 확인(Acknowledge) 처리가 필요합니다.

레벨용도확인 필요사용 기준
TRACE 개발/디버깅 추적 ❌ 개발 환경에서만 사용 (현재 미사용)
DEBUG 디버깅 정보 ❌ 개발 환경에서만 사용 (현재 미사용)
INFO 정상 동작 기록 ❌ 모든 성공 작업, 페이지 방문, 정상 이벤트
WARN 경고/거부 ❌ 권한 거부, 비즈니스 규칙 차단, 비정상 시도
ERROR 오류 발생 ✅ DB/RPC 실패, JavaScript 예외, API 오류
FATAL 치명적 오류 ✅ 서버 연결 불가 등 시스템 장애
CRITICAL 긴급 오류 ✅ 예약됨 (현재 미사용)

🏷️ 액션 네이밍 규칙

패턴레벨의미예시
ACTION_NAME INFO 성공한 작업 USER_CREATE, TALENT_GIVE, ORDER_REQUEST_SUCCESS
*_DENIED WARN 권한/규칙에 의한 거부 USER_CREATE_DENIED, TALENT_GIVE_DENIED
*_FAIL ERROR DB/RPC 오류 반환 PASSWORD_CHANGE_FAIL, PURCHASE_LOAD_FAIL
*_ERROR ERROR JavaScript 예외 발생 LOGIN_ERROR, PASSWORD_CHANGE_ERROR

🤖 자동 로깅 항목

코드 호출 없이 자동으로 기록되는 항목들입니다.

액션레벨트리거기록 데이터
PAGE_VIEW INFO 모든 페이지 로드 시 autoLogPageView() 페이지 경로, 사용자명
JS_ERROR ERROR window.onerror (캐치되지 않은 JS 에러) 에러 메시지, 파일명, 라인/컬럼
PROMISE_REJECTION ERROR unhandledrejection 이벤트 rejection reason
CONNECTION_FAIL FATAL 메인 페이지 Supabase 연결 실패 에러 메시지
PAGE_VIEW: v3.40.0부터 페이지 조회 로그(PAGE_VIEW) 기록이 비활성화되었습니다. autoLogPageView() 호출은 유지되지만 activity_logs에 저장되지 않습니다.
Slack 알림: WARN, ERROR, FATAL, CRITICAL 로그는 js/activity-log.js에서 log_alert로 Edge Function slack-notify를 호출합니다. 자세한 라우팅은 운영 > Slack 알림 룰에서 확인합니다.

⚙️ writeLog 함수 구조

writeLog(level, action, page, details)
// level: 'TRACE'|'DEBUG'|'INFO'|'WARN'|'ERROR'|'FATAL'|'CRITICAL'
// action: 작업 식별자 (예: 'USER_CREATE')
// page: 페이지 경로 (null이면 자동 감지)
// details: 추가 정보 객체

편의 함수

logInfo('TALENT_GIVE', { userId, 금액: amount, description })
logWarn('TALENT_GIVE_DENIED', { userId, 사유: '권한 부족' })
logError('TALENT_GIVE_FAIL', { userId, error: err.message })

📝 참고 사항

저장소: 모든 로그는 Supabase activity_logs 테이블에 저장됩니다.
DB 권한: 로그가 DB에 쌓이지 않으면 docs/TASK-047_activity_logs_grants.sql로 activity_logs INSERT 권한과 RLS 정책을 복구합니다.
소프트 삭제: 로그는 물리적으로 삭제되지 않고 is_deleted=true로 표시됩니다.
한글 라벨: 액션 라벨은 js/codes.js 기본 코드북과 DB activity_logs.action 코드 그룹을 우선 사용합니다. writeLog() 호출 시 자동으로 details._actionLabel에 한글 라벨이 추가되어 기존 로그도 하위호환 표시됩니다.
로그 관리 페이지: 운영 > 로그에서 조회/확인 처리/삭제 가능. 페이지당 항목 수 콤보 박스는 필터 영역 가장 아래의 삭제 대기 목록 줄 오른쪽에 배치되며, 선택값은 계정별 설정으로 저장됩니다.
Slack 알림 룰: 운영 > Slack 알림 룰에서 구매/가입/부서 이동/Q&A/로그 알림 라우팅을 확인 가능
