**작업 브랜치:** develop

## 작업 개요

- **작업명:** 관리자 인증 시스템 및 보고서/로그 관리 페이지 구축
- **목적:** 관리자 전용 페이지를 통해 작업 보고서 열람 및 활동 로그를 관리하고, 향후 모든 수정 작업에 적용할 표준 프로세스를 체계화

## 구현 범위

### Supabase 테이블 (3개)
- admin_users: 관리자 계정 (SHA-256 해시, RPC로만 접근)
- reports: 작업 보고서 (4종 유형)
- activity_logs: 활동 로그 (7레벨, ERROR+ 필수 확인)

### 관리자 페이지 (5개)
- login.html: 로그인
- index.html: 대시보드 (요약 카드, ERROR+ 배지)
- reports.html: 보고서 열람 (유형별 필터, 상세 모달)
- logs.html: 로그 관리 (레벨 필터, 확인 모달, 페이지네이션)
- change-password.html: 비밀번호 변경 (첫 로그인 강제)

### 공통 모듈
- auth.js: 인증 (SHA-256, RPC 기반)
- activity-log.js: 로그 시스템 (7레벨, 글로벌 에러 핸들러)
- KST 유틸리티: toKST(), formatKST(), formatKSTShort()

### 로그 레벨 체계
- TRACE, DEBUG: 개발/디버깅
- INFO: 일반 동작
- WARN: 경고
- ERROR, FATAL, CRITICAL: 에러 (필수 확인)
