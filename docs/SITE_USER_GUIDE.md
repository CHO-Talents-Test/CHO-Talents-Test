# ⭐ 달란트 마을 사이트 안내서

이 문서는 개발자가 아닌 일반 사용자가 `CHO-Talents` 사이트를 쉽게 이용할 수 있도록 정리한 안내서입니다. 사이트의 `소개` 메뉴에는 단일 `가이드` 항목이 표시되며, 로그인 상태와 권한에 맞는 역할별 가이드가 자동으로 열립니다. 비로그인 사용자는 학생 가이드로 이동합니다.

**사이트 주소:** https://cho-talents.github.io/CHO-Talents/

**문서 기준 버전:** v3.60.0

**작업 전 규칙:** `.cursor/rules/00-workflow-guardrails.mdc`를 먼저 읽고, 아래 최신 가이드 페이지와 함께 기준을 맞춘다.

- `https://cho-talents.github.io/CHO-Talents/teacher-guide.html`
- `https://cho-talents.github.io/CHO-Talents/admin-guide.html`
- `https://cho-talents.github.io/CHO-Talents/docs/page-permission-rules.html`

## 1. 달란트 마을에서 할 수 있는 일

| 사용자 | 할 수 있는 일 |
|---|---|
| 학생 | 내 달란트 잔액과 적립/사용 내역 확인, 학생용 상점 상품 확인, 상품 구매 신청, 내 구매 상품 확인, 달란트 QR 수령(카메라 스캔), Q&A 질문/답변 |
| 교사 | 내 달란트 확인, 교사용/학생용 상점 상품 확인, 상품 구매 신청, 권한 범위 내 학생 대리 구매 |
| 부서 담당 교사 | 담당 부서 학생/교사 관리, 달란트 지급/사용 처리, 상품 관리, 담당 부서 구매 신청 처리, Q&A 답변 |
| 구매 담당 교사 | 부서 담당 교사와 동일한 메뉴 접근, 구매 관리에서 모든 부서 주문 처리 가능 |
| 부장 교사/전도사님/관리자 | 부서, 사용자, 관리자, 상품, 구매, 공지, 보고서, 버전, 로그 등 운영 관리 (부장 교사는 사용자/관리자 관리에서 조회만 가능) |
| 비로그인 사용자 | 메인 화면, 가이드(학생 가이드), 달란트 적립 안내, Q&A FAQ, 학생용 상점, 계정 등록 신청 |

## 2. 처음 이용하는 방법

1. 사이트에 접속합니다.
2. 계정이 있으면 `로그인`을 누릅니다.
3. 계정이 없으면 `계정 등록 신청`을 누릅니다.
4. 아이디 중복확인, 이름, 소속 부서를 입력하고 신청합니다.
   - 아이디는 영문/숫자/`_`/`-`만 사용할 수 있습니다 (한글 불가).
5. 관리자가 승인하면 초기 비밀번호 `1234`로 로그인할 수 있습니다.
6. 첫 로그인 때는 반드시 새 비밀번호로 변경해야 합니다.

## 3. 로그인 후 이동하는 화면

로그인하면 모든 사용자가 먼저 `메인` 화면으로 이동합니다. 이후 상단 메뉴에서 내 권한에 맞는 기능만 볼 수 있습니다.

운영자가 활성화한 공지가 있으면 메인 화면에 공지 팝업이 표시됩니다. `닫기`를 누르면 이번 접속에서만 닫히고, `다시 열지 않음`을 누르면 해당 계정에는 그 공지가 다시 표시되지 않습니다.

첫 로그인 상태라면 어떤 권한이든 먼저 `비밀번호 변경` 화면으로 이동합니다. 새 비밀번호로 변경한 뒤 다시 메인 화면과 권한별 메뉴를 사용할 수 있습니다.

상단 네비게이션에는 `⭐ 달란트 마을` 브랜드와 함께 소개, 달란트, 상품, 관리, 운영 메뉴가 표시됩니다. 로그인 상태와 권한에 따라 메뉴 항목이 자동으로 달라집니다.

부서 담당 교사 이상에게는 상단 메뉴에 처리 필요 건수 배지가 표시됩니다. `관리` 배지는 처리 가능한 가입 신청/부서 이동 요청, `상품` 배지는 처리 가능한 구매 신청, `운영` 배지는 미확인 ERROR 이상 로그, `소개` 배지는 Q&A 미답변 질문 건수를 의미합니다. 구매 담당 교사는 구매 관리와 상품 배지에서 전체 부서 주문이 집계됩니다.

권한명, 사용자 유형, 상품 카테고리, 구매 상태, QR 반복 유형, 로그/작업 이력 유형 같은 구분값은 v3.50.0부터 공통 코드 기준으로 표시됩니다. 같은 상태나 카테고리는 상점, 내 구매 상품, 구매 관리, 구매 통계, 작업 이력에서 동일한 이름과 순서로 보입니다.

### 역할별 가이드

| 가이드 | 접근 권한 | 내용 |
|---|---|---|
| 학생 가이드 | 공개 | 계정 신청, 달란트 확인, 상품 구매, QR 수령 |
| 교사 가이드 | 일반 교사(40+) | 담당 부서/반 학생 달란트 지급, 교사용 상품, 대리 구매 |
| 부서 담당 교사 가이드 | 부서 담당 교사(60+) | 일반 교사가 없는 부서에서 해야 할 일, 담당 부서 사용자/구매/Q&A 관리 |
| 구매 담당 교사 가이드 | 구매 담당 교사(70+) | 전체 부서 구매 신청 처리, 구매 통계, Slack 구매 알림 기준 |
| 부장 교사 가이드 | 부장 교사(80+) | 학생 일괄 등록, 달란트 반환, 보고서/버전, 운영 룰 문서 |
| 전도사님 가이드 | 전도사님(90+) | 달란트 항목, QR 관리, 공지 관리, 상품 삭제, Q&A 삭제, 부서 비활성화 |
| 관리자 가이드 | 부서 담당 교사(60+) 이상 | 전체 운영 메뉴와 관리자 전용 설정 요약 |

상단 `소개 > 가이드`를 누르면 현재 계정 기준으로 위 표의 적절한 가이드가 열립니다. 예를 들어 구매 담당 교사는 구매 담당 교사 가이드, 전도사님은 전도사님 가이드, 관리자 또는 최고 관리자는 관리자 가이드로 연결됩니다.

### Slack 알림 (채널별 분리)

부서 담당 교사 이상 운영자는 사이트에 접속하지 않아도 담당 부서 Slack 채널에서 주요 운영 이벤트 알림을 받을 수 있습니다. 알림은 이벤트 유형과 대상 부서에 따라 각각 다른 채널로 전송됩니다.

| 알림 | 언제 보내나요? | 어디로 보내나요? | 알림에 포함되는 정보 |
|---|---|---|---|
| 🛒 신규 구매 신청 | 학생/교사가 상품 구매를 신청하거나, 교사가 대리 구매를 신청할 때 | 신청자 소속 부서 채널 | 신청자, 상품명, 금액, 구매 유형, 소속 부서 |
| 📦 구매 상태 변경 | 구매 관리에서 '구매 신청 → 상품 준비' 단계로 전환할 때 | 상품 관리 채널 | 상품명, 신청자, 이전/변경 상태, 처리자 (일괄 시 건수) |
| 📝 신규 가입 신청 | 계정 등록 신청 페이지에서 신청이 제출될 때 | 신청 부서 채널 | 아이디, 이름, 소속 부서 |
| 🔄 부서 이동 신청 | 사용자 관리에서 부서 이동 요청이 생성될 때 (90등급 미만 즉시 이동 제외) | 이동 대상 부서 채널 | 대상자, 이전/이동 부서, 사유, 신청자 |
| ⚠️ WARN+ 로그 | 사이트에서 경고(`WARN`) 이상 로그가 기록될 때 | 운영 채널 | 로그 레벨, 액션, 페이지, 상세 내용 |
| ❓ Q&A 질문 | Q&A 페이지에 새 질문이 등록될 때 | Q&A 채널 | 등록자, 질문 내용 |

Slack 알림은 사이트 네비게이션 배지와 함께 참고하면 처리가 필요한 업무를 더 빠르게 파악하는 데 도움이 됩니다. 각 부서 담당자는 자기 부서 채널만 확인하면 됩니다.

### 메인 페이지 즐겨찾기
로그인 후 메인 페이지의 바로가기 카드를 개인 설정할 수 있습니다.
- `⭐ 즐겨찾기 설정` 버튼을 누르면 표시할 바로가기를 선택할 수 있습니다.
- PC와 모바일 모두 최소 1개, 최대 10개까지 선택 가능합니다.
- 내 권한에서 실제로 보이는 메뉴만 즐겨찾기 후보에 나타나며, 권한이 없는 항목은 저장되어 있어도 자동으로 제외됩니다.
- 설정은 데이터베이스에 저장되므로 다른 기기에서 로그인해도 동일하게 표시됩니다.
- 비로그인 사용자는 기본 3개 카드(달란트 적립/상품 구매/내 달란트)가 표시됩니다.

### 테마 설정
모든 페이지 네비게이션 바에서 테마를 변경할 수 있습니다.
- 네비게이션 바 우측의 테마 스위치로 `일반`과 `다크` 모드를 전환할 수 있습니다.
- **일반 모드**: 기본 보라색 테마
- **다크 모드**: 어두운 배경과 어두운 드롭다운/모달/카드/표면을 사용하는 테마. 가이드, Q&A, QR, 로그/작업 이력 룰, 버전 이력 같은 안내 페이지도 어두운 표면색으로 표시됩니다.
- 테마 설정은 로그인 계정에 DB로 저장되어 다른 환경에서도 로그인 시 동일하게 적용됩니다.
- 로그아웃하면 기본(일반) 테마로 돌아갑니다.
- 비로그인 상태에서도 localStorage에 저장되어 같은 브라우저에서는 유지됩니다.

### 페이지당 항목 수 설정
목록형 그리드 화면(사용자 관리, 달란트 관리, 달란트 관리 상세 이력, 구매 관리, 구매 통계, 로그, QR 목록/수령자 팝업, 내 구매 상품, 달란트 수령 최근 내역, 부서 관리 등)마다 **페이지당 항목 수** 콤보 박스가 표시됩니다.
- 3~30개 범위에서 원하는 개수를 선택할 수 있습니다.
- 선택한 값은 계정별로 DB(`user_preferences.page_sizes`)에 저장되어 다른 기기에서도 동일하게 적용됩니다.
- 기본값은 PC 20개, 모바일 10개입니다.

## 4. 학생 이용 안내

학생은 주로 아래 메뉴를 사용합니다.

| 메뉴 | 설명 |
|---|---|
| 달란트 적립 | 어떤 활동으로 달란트를 받을 수 있는지 확인합니다. 각 카드에는 활성 달란트 항목의 지급 규칙, 설명, 지급 수량(`+N 달란트`)이 표시됩니다. |
| 상품 구매 | 학생용 달란트 상점 상품과 필요한 달란트를 확인합니다. 로그인하면 구매 신청을 할 수 있습니다. |
| 내 달란트 | 사용 가능 달란트, 상품 수령 예정, 사용 대기 달란트, 사용 완료 달란트, **반환 달란트**, **예외 달란트**, 누적 적립 달란트, 최근 내역을 확인합니다. 반환은 상품 구매 사용과 별도로 표시되고, 예외 지급은 별도 배지로 구분됩니다. |
| 내 구매 상품 | 내가 신청한 상품의 진행 상태를 확인합니다. |
| Q & A | 자주 묻는 질문을 보고, 로그인 후 궁금한 내용을 질문할 수 있습니다. |

`내 달란트` 화면에서 지급 취소 이력의 달란트 트랜잭션 ID는 표시되지 않습니다.

### 상품 구매하기

1. `상품 구매` 페이지에서 원하는 상품의 `구매 신청` 버튼을 누릅니다.
2. 확인 창에서 `확인`을 누르면 구매 신청이 완료됩니다.
3. 신청한 달란트는 바로 차감되지 않고 `사용 대기` 상태로 표시됩니다.
4. `내 달란트`에서 `사용 대기 달란트`와 실제 `사용 가능 달란트`를 확인할 수 있습니다.
5. `내 구매 상품`에서 신청한 상품의 진행 상태를 확인할 수 있습니다.
6. 선생님이 상품 구매를 확정하면 실제로 달란트가 차감되고, 지급 완료 상태까지 확인할 수 있습니다.

구매 진행 단계: `구매 신청` → `상품 준비` → `상품 구매 확정(달란트 차감)` → `상품 지급`

`구매 신청` 단계에서는 취소 버튼을 눌러 구매를 취소할 수 있습니다. 취소하면 대기 중이던 달란트가 복원되고, 해당 주문은 `구매 취소` 상태로 변경됩니다. 취소된 주문은 내 구매 상품 목록에 표시되지 않습니다. 취소 시 실제 주문 상품 ID 기준으로 정확하게 대상 건이 취소됩니다.

## 5. 교사 이용 안내

교사는 `내 달란트`에서 본인 달란트 내역을 확인할 수 있습니다. 사용 완료, 반환 달란트, 예외 달란트가 별도 요약 박스로 표시되며, 내역 유형도 적립·예외·사용·반환으로 구분됩니다. 지급 취소 이력에서 달란트 트랜잭션 ID는 표시되지 않습니다.

`상품 구매`에서는 교사용 상품과 학생용 상품을 탭으로 나누어 볼 수 있습니다. 교사로 로그인하면 교사용 탭이 기본으로 선택됩니다.

교사는 **교사용 상품만** 직접 구매 신청할 수 있으며, 학생용 상품은 조회만 가능합니다(대리 구매를 위해 가격과 재고 확인).

일반 교사는 담당 부서/반의 학생 달란트 지급/사용 처리가 가능합니다. 권한이 있으면 `상품 구매` 화면에서 `대리 구매` 기능으로 담당 범위 안의 학생을 선택해 학생용 상품을 구매 신청할 수 있습니다. 대리 구매 시 선택한 사용자의 유형(학생/교사)에 맞는 상품만 표시됩니다.

카테고리 필터와 상품명/카테고리 검색으로 원하는 상품을 빠르게 찾을 수 있습니다. 카테고리명은 운영 코드 기준으로 관리되어 상품 구매 화면과 상품 관리 화면에서 동일하게 표시됩니다.

## 6. 운영자 이용 안내

부서 담당 교사 이상은 상단 메뉴에 관리 기능이 추가로 보입니다. 보이는 메뉴는 권한에 따라 다릅니다.

상단 배지는 현재 계정의 권한 범위 안에서 실제로 처리할 수 있는 건수를 기준으로 표시됩니다.

목록형 화면은 공통 페이징 규칙을 사용합니다. PC에서는 한 페이지에 20개, 모바일에서는 10개씩 표시되며, 현재 페이지 번호가 강조 표시됩니다. 페이지가 7개를 초과하면 말줄임표(...)로 축약됩니다. 예를 들어 8페이지 중 4페이지를 보는 경우 `1 ... 3 4 5 6 7 8` 형태로 표시됩니다. 각 그리드 상단 또는 모달 이력 상단의 **페이지당 항목 수** 콤보 박스(3~30개)로 개인별 표시 개수를 변경할 수 있으며, 설정은 DB에 저장됩니다.

### 그리드 헤더 클릭 정렬
목록형 화면의 테이블 헤더를 클릭하면 해당 열 기준으로 오름차순/내림차순 정렬이 토글됩니다.
- 정렬 중인 열에는 ▲(오름차순) 또는 ▼(내림차순) 화살표가 표시됩니다.
- 정렬 가능한 열에는 ⇅ 아이콘이 희미하게 표시됩니다.
- 적용 페이지: 사용자 관리, 관리자 관리, 달란트 관리, 달란트 통계, 구매 관리, 상품 관리, 부서 소속보기 모달
- 기본 정렬 순서:
  - 사용자 목록/달란트 관리: 부서 → 반(null 마지막) → 이름
  - 관리자 목록: 권한 등급(내림차순) → 부서 → 이름
  - 부서 목록: 부서명
  - 달란트 통계 부서별: 달란트(내림차순) → 인원 → 항목 → 부서
  - 달란트 통계 사용자별: 달란트(내림차순) → 항목 → 부서 → 이름
  - 구매 통계 부서별: 부서
  - 구매 통계 사용자별: 부서 → 개수(내림차순) → 이름
  - 구매 통계 유형별: 상품 → 상태
  - 구매 관리: 상태(단계순) → 신청일(내림차순) → 상품명 → 부서 → 신청자
  - 상품 관리: 카테고리 → 상품명 → 가격(내림차순)

### 부서 담당 교사 사용자 관리 권한
부서 담당 교사(rank 60+)는 담당 부서 소속 학생에 대해 다음 작업이 가능합니다.
- **반 변경**: 사용자 수정 모달에서 반(class_number) 수정
- **비밀번호 초기화**: 관리 드롭다운에서 비밀번호를 `1234`로 초기화

다음 작업은 **부장 교사(80+) 이상**만 가능합니다.
- 사용자 **삭제**
- **부서 이동** 요청/처리 (60~79는 담당 부서 범위 내 수정·비밀번호 초기화만 가능)

### Super Admin (최고관리자) 특수 권한
`is_super_admin = true`인 계정은 사용자 관리에서 다음 제약이 모두 해제됩니다.
- 모든 사용자 관리 가능 (다른 super admin 포함)
- 수정 모달에서 **역할**, **권한 레벨**, **소속 부서**, **담당 관리 부서** 모두 편집 가능
- `buildPermOptions`에서 모든 권한 옵션(학생~관리자) 제한 없이 선택 가능
- 서버 사이드 RPC(`admin_update_user`)에서도 super admin은 모든 레벨 할당 가능

### 달란트 지급하기

1. `달란트 관리`에서 학생/교사 탭을 선택합니다.
2. 대상 사용자를 선택합니다.
3. **출석 버튼**: 각 사용자 행의 `출석` 버튼을 클릭하면 출석 달란트가 즉시 지급됩니다.
   - 이번 주(월~일) 이미 지급된 경우 `✓출석 취소` 버튼으로 변경되며, 클릭 시 지급이 취소됩니다.
   - 달란트 지급은 주간(월~일) 1회로 제한됩니다. 월요일이 되면 자동으로 초기화됩니다.
   - 본인에게는 달란트를 지급할 수 없습니다.
4. **달란트 지급 모달**: 항목별로 지급/취소가 가능합니다. 이미 지급된 항목에는 `취소` 버튼이 표시됩니다.
5. **항목 선택 지급**: `달란트 지급` 버튼을 눌러 지급 항목 체크박스를 선택합니다.
   - 이미 이번 주(월~일) 지급된 항목은 자동 표시됩니다.
6. 전도사님(90+) 이상은 `이번 주 제한 예외 지급`을 선택하고 사유를 입력하면, 이미 이번 주 지급된 항목도 다시 선택해 예외 지급할 수 있습니다.
7. `지급 확정` 또는 `예외 지급 확정` 버튼을 누르면 선택한 항목이 일괄로 지급됩니다.
8. 수동으로 금액과 사유를 직접 입력하여 적립/사용할 수도 있습니다.

### 달란트 반환

부장 교사 이상 권한자는 이미 지급된 달란트를 반환할 수 있습니다.

1. 대상 사용자의 `상세` 버튼을 눌러 달란트 내역을 확인합니다.
2. 적립(earn) 항목 중 아직 반환되지 않은 항목에 `반환` 버튼이 표시됩니다.
3. 반환 사유를 입력하고 반환 처리합니다.
4. 동일 항목은 1회만 반환 가능합니다. 반환된 항목은 "반환됨" 표시로 변경됩니다.
5. 달란트 지급 모달에서 `취소` 버튼으로 처리한 항목과 상세에서 `반환` 처리한 항목은 동기화됩니다.
6. 단, 잔여 달란트가 0보다 작아질 수 없습니다.

반환 처리된 달란트는 사용자 `내 달란트` 화면에서 **사용 완료**와 구분됩니다. 요약 카드에 **반환 달란트** 박스가 별도로 표시되고, 달란트 내역 테이블에서는 **적립**·**사용**·**반환** 세 가지 유형으로 구분해 볼 수 있습니다.

### 부서 이동

사용자의 소속 부서를 변경하려면 `사용자 관리`의 `부서 이동` 기능을 사용합니다.

- 전도사님 이상: 즉시 이동 또는 전체 요청 처리 가능
- 부장 교사: 전체 요청을 볼 수 있고, 담당 관리 부서 요청을 처리 가능
- 부서 담당 교사: 담당 부서 범위에서 요청/처리 가능

### 달란트 QR 관리

전도사님(90등급) 이상이 QR 코드를 생성하고 관리할 수 있습니다.

1. `달란트 QR 관리` 페이지에서 **지급 대상**(학생/교사)을 선택합니다. 선택에 따라 해당 대상에게 지급 가능한 달란트 항목만 표시됩니다.
2. 달란트 항목을 선택하거나, "직접 입력"으로 설명과 금액을 입력합니다.
3. **지급 방식**을 선택합니다: 무제한 또는 선착순(인원수 지정).
4. **반복 수령 가능 여부**를 설정합니다:
   - `X`: 1회만 수령 가능 (기본값)
   - `O`: 반복 수령 가능 → 반복 방식 선택:
     - `매일`: 매일 00:00에 초기화되어 다시 수령 가능
     - `요일 반복`: 지정한 요일(다중 선택)에만 수령 가능 (매일 초기화)
     - `주차+요일 반복`: 매월 지정 주차의 지정 요일에만 수령 가능 (매일 초기화)
5. **유효 기간**을 라디오 버튼으로 선택합니다:
   - `지정일`: 선택한 날짜와 유효 시간(from~to)에만 수령 가능 (기본 00:00 ~ 23:59)
   - `기간`: from ~ to 기간 동안 유효
   - `무기한`: 기간 제한 없이 사용 가능
6. **위치 제한**을 설정할 수 있습니다:
   - `제한 없음`: 어디서나 수령 가능
   - `특정 위치`: 카카오맵으로 주소를 검색/선택하고, 반경(100m~5km)을 지정합니다. 기본값은 500m입니다.
   - 위치 제한 QR을 수령하는 사용자의 기기 또는 브라우저에서 위치 정보 제공이 차단되어 있으면 알림창과 화면 메시지가 표시되며, 위치 권한을 허용한 뒤 다시 시도해야 합니다.
7. `🎫 QR 코드 생성` 버튼을 눌러 QR 코드를 생성합니다.
8. 생성된 QR 코드는 2차원 이미지로 표시됩니다. `인쇄` 버튼으로 QR 이미지를 출력할 수 있습니다.
9. `수정` 버튼으로 대상, 항목, 반복 수령, 유효 기간, 위치 제한을 변경할 수 있으며, 저장 시 새로운 QR 코드가 생성됩니다.
10. QR 리스트에서 **검색**(설명/코드) 및 **필터**(지급 대상, 제한 조건, 생성일)로 원하는 코드를 빠르게 찾을 수 있습니다.
11. 계정 유형(학생/교사)이 QR 대상과 일치하지 않으면 수령이 차단됩니다.
    달란트 수령 화면에서 카메라로 QR을 읽은 결과 메시지는 카메라 영역 위에 표시됩니다.
12. `👥 수령자` 버튼 클릭 시 수령자 팝업에 이름/부서/수령 시간/반환 상태가 페이징과 함께 표시됩니다. 표시 개수(3~30개)는 콤보박스로 변경하며 사용자별로 DB에 저장됩니다.

### 구매 관리

구매 관리 페이지에서 상품 구매 신청을 처리합니다. 모든 상태 탭(전체/구매신청/상품준비/상품구매/상품지급)에서 날짜(from~to), 부서 필터, 기간 프리셋(오늘/1주/1달/1년)으로 원하는 주문만 조회할 수 있습니다. 구매 담당 교사는 부서 필터와 처리 권한이 전체 부서 범위로 적용됩니다. 구매 상태 이름, 색상, 표시 순서는 공통 코드 기준이라 `내 구매 상품`, `구매 관리`, `구매 통계`에서 동일하게 보입니다.

| 단계 | 설명 |
|---|---|
| 구매 신청 | 학생/교사가 상점에서 신청. 달란트는 아직 차감되지 않음 |
| 상품 준비 | 담당 교사가 신청을 확인하고 상품 준비 상태로 변경 |
| 상품 구매 | 처리 권한자가 실제 구매를 확정. 이때 달란트가 차감됨 |
| 상품 지급 | 상품을 실제로 전달한 후 지급 완료 처리 (일괄 처리 가능) |

## 7. 달란트는 어떻게 반영되나요?

1. 학생이나 교사가 활동을 합니다.
2. 선생님 또는 운영자가 `달란트 관리`에서 대상자를 선택합니다.
3. 지급 항목의 체크박스를 선택하고 `지급 확정`을 누르거나, 사유와 금액을 직접 입력합니다.
4. 처리 후 사용자의 `내 달란트` 화면에 잔액과 내역이 표시됩니다.
5. 상품 교환 시에는 `상품 구매` 페이지에서 구매 신청을 하면 운영자가 처리합니다.

## 8. 내 달란트 화면 보는 방법

지급 취소 이력에서 달란트 트랜잭션 ID는 표시되지 않습니다.

상단 요약 카드에는 사용 가능 달란트, 수령 대기 상품수, 사용 대기 달란트, **사용 완료 달란트**, **반환 달란트**, **예외 달란트**, 누적 적립 달란트가 표시됩니다. **반환 달란트**는 운영자가 지급을 회수한 금액을 모아 보여 주는 별도 박스이며, **예외 달란트**는 주간 제한을 예외로 다시 지급된 금액입니다. **사용 완료 달란트**에는 실제 상품 구매 등에 사용된 달란트만 포함됩니다.

| 항목 | 의미 |
|---|---|
| 누적 적립 달란트 | 지금까지 받은 달란트 합계 |
| 사용 완료 달란트 | 실제 상품 구매 등에 사용 처리된 달란트 (반환은 포함하지 않음) |
| 반환 달란트 | 운영자가 지급을 반환한 달란트 합계 (사용과 별도 표시) |
| 예외 달란트 | 운영자가 주간 제한을 예외로 다시 지급한 달란트 합계 |
| 사용 대기 달란트 | 구매 신청했지만 아직 확정되지 않은 달란트 |
| 사용 가능 달란트 | 실제로 사용할 수 있는 잔여 달란트 (잔여 - 사용 대기) |

**달란트 내역** 테이블의 **유형** 열에는 **적립**, **예외**, **사용**, **반환** 네 가지가 표시됩니다. 예외 지급은 노란색 **예외** 배지로, 반환 건은 분홍색 **반환** 배지로 구분되며, 설명에는 반환 사유만 표시됩니다.

`내 구매 상품` 화면에서는 내가 신청한 상품의 현재 단계를 확인할 수 있습니다. 단계는 `구매 신청`, `상품 준비`, `상품 구매`, `상품 지급` 순서로 표시됩니다. `구매 신청` 단계의 주문은 취소할 수 있으며, 취소된 주문은 목록에서 자동으로 숨겨집니다.

## 9. 비밀번호와 계정

### 세션 관리

로그인 상태는 **24시간 유휴** 시 자동으로 만료됩니다.

- 페이지 조회, 클릭, 스크롤, 키 입력 등 사이트를 이용하는 활동이 있으면 세션 만료 시간이 다시 24시간으로 초기화됩니다.
- 다른 브라우저 탭으로 이동했다가 돌아올 때도 세션 만료 여부를 다시 확인합니다. 만료된 경우 안내 후 로그인 화면으로 이동합니다.
- 장시간 탭을 열어 두었다가 다시 사용할 때는 로그인이 필요할 수 있습니다.
- 세션 만료나 권한 부족으로 보호 페이지에서 다른 화면으로 이동한 경우 운영 로그에 원인, 요청 페이지, 필요/실제 권한, 이동 대상이 기록되어 문제 원인을 추적할 수 있습니다.

비밀번호와 계정 관련 안내는 다음과 같습니다.

- 가입 승인 후 초기 비밀번호는 `1234`입니다.
- 첫 로그인 시 새 비밀번호로 변경해야 합니다.
- **비밀번호 규칙**: 8자 이상, 영문과 숫자를 반드시 포함해야 합니다.
- 초기 비밀번호 `1234`는 새 비밀번호로 사용할 수 없습니다.
- 비밀번호를 잊은 경우 담당 선생님이나 관리자에게 초기화를 요청하세요.
- 초기화 후 다시 `1234`로 로그인하고 새 비밀번호로 변경하면 됩니다.

## 10. Q&A와 자주 묻는 질문

자주 묻는 질문은 [Q & A 페이지](https://cho-talents.github.io/CHO-Talents/qna.html)에서 확인할 수 있습니다.

- 비로그인 사용자는 공개 FAQ를 볼 수 있고, 이름을 입력하여 질문 등록도 가능합니다.
- 로그인 사용자는 새 질문을 등록할 수 있습니다.
- 부서 담당 교사 이상은 질문에 댓글(답변)을 달 수 있고, 필요한 질문을 FAQ로 등록할 수 있습니다.
- 관리자는 FAQ 항목을 직접 추가할 수 있습니다 (FAQ 직접 등록 버튼).
- 전도사님(90+) 이상은 등록된 질문을 삭제할 수 있습니다.

| 질문 | 답변 |
|---|---|
| 로그인이 안 돼요. | 아이디와 비밀번호를 다시 확인하세요. 승인 전 계정은 로그인할 수 없으며, "승인 대기 중" 안내가 표시됩니다. |
| 계정 신청 후 바로 사용할 수 있나요? | 아니요. 관리자가 승인한 뒤 사용할 수 있습니다. |
| 상품은 어떻게 구매하나요? | 로그인 후 `상품 구매` 페이지에서 `구매 신청` 버튼을 누르세요. 이후 선생님이 처리합니다. |
| 구매 신청했는데 달란트가 안 줄었어요. | 정상입니다. 구매 신청 시에는 `사용 대기`로 표시되고, 선생님이 상품 구매를 확정하면 실제로 차감됩니다. |
| 내 달란트가 맞지 않는 것 같아요. | 담당 선생님에게 문의해 최근 적립/사용/반환 내역을 확인하세요. |
| 교사용 상품이 안 보여요. | 교사 계정으로 로그인해야 교사용 상품 탭을 볼 수 있습니다. |
| 메뉴가 사람마다 달라요. | 사이트 권한에 따라 사용할 수 있는 메뉴만 보입니다. |
| 오류 메시지가 표시돼요. | 에러는 한글로 안내됩니다. 같은 오류가 반복되면 관리자에게 문의하세요. |
| 비밀번호를 잊어버렸어요. | 담당 선생님에게 초기화를 요청하세요. 초기화 후 1234로 로그인하고 새 비밀번호로 변경하면 됩니다. |

## 11. 네비게이션 메뉴 구조

사이트 상단 네비게이션은 메뉴 그룹 드롭다운 방식으로 구성되어 있습니다. 각 메뉴 그룹을 클릭하면 하위 메뉴가 펼쳐지고, 다른 곳을 클릭하면 닫힙니다. 데스크탑에서는 마우스를 올려도(hover) 메뉴가 열립니다. 우측 테마 스위치로 일반/다크 모드를 바로 전환할 수 있습니다.

### 모바일 화면 (768px 이하)

화면 너비가 768px 이하인 모바일에서는 상단 바 배치가 다음과 같이 정리됩니다.

- **좌측:** `⭐ 달란트 마을` 브랜드(메인으로 이동)
- **우측(왼쪽부터):** 테마 스위치 → 햄버거 메뉴(☰) → 로그아웃 → (권한 이모지) 이름

햄버거 메뉴를 누르면 소개·달란트·상품·관리·운영 메뉴 그룹이 아래로 펼쳐집니다. 모바일에서는 hover로 메뉴가 열리지 않으며, 그룹 이름을 탭해야 하위 메뉴를 볼 수 있습니다.

| 그룹 | 메뉴 항목 | 접근 권한 |
|------|-----------|-----------|
| 소개 | 가이드, 달란트 적립, Q & A | 가이드는 단일 메뉴로 표시되며 비로그인은 학생 가이드, 로그인 사용자는 권한에 맞는 역할별 가이드로 연결 |
| 달란트 | 내 달란트, 달란트 관리, 예외 지급/반환 관리 | 로그인 사용자 / 일반 교사(40)+ / 부서 담당 교사(60)+ |
| 상품 | 상품 구매, 내 구매 상품, 상품 관리, 구매 관리, 구매 통계 | 모든 사용자 / 로그인 사용자 / 부서 담당 교사(60)+ |
| 관리 | 사용자 관리, 학생 일괄 등록, 부서 관리, 질문 관리, 대시보드, 관리자 관리 | 사용자/부서/질문/대시보드는 부서 담당 교사(60)+, 학생 일괄 등록/관리자 관리는 부장 교사(80)+ |
| 운영 | 페이지 권한 룰, 로그 작성 룰, Slack 알림 룰, 작업 이력 작성 룰, 공지 관리, 버전, 보고서, 페이지 접근, 페이지 기능, 작업 이력, 로그 | 룰 문서/보고서/버전은 부장 교사(80)+, 공지 관리는 전도사님(90)+, 페이지 접근/기능/작업 이력/로그는 관리자(100)+ |

## 12. 개인정보와 이용 기록

사이트 운영과 오류 확인을 위해 로그인, 오류 발생, 인증/권한 리디렉트, 관리 작업 같은 활동 기록이 `activity_logs`에 저장될 수 있습니다. **v3.40.0부터 페이지 조회(PAGE_VIEW) 로그는 기록되지 않습니다.** 작업 이력은 이 로그 중 관리 작업만 필터링해 보여줍니다. 로그 action은 코드 기준 라벨과 함께 저장되어 운영 화면에서 한글 작업명으로 표시됩니다. 오류 확인용 로그에는 브라우저, 기기 종류, 화면 크기, IP 등 접속 환경 정보가 함께 남을 수 있습니다. 공지의 `다시 열지 않음` 선택은 계정별 공지 숨김 상태로 저장됩니다.

비밀번호는 사이트 화면에 표시되지 않으며, 담당자도 기존 비밀번호를 확인할 수 없습니다. 필요한 경우 새 초기 비밀번호로 재설정만 할 수 있습니다.

다른 사용자의 아이디는 관리자를 제외하고 화면에 표시되지 않습니다.
