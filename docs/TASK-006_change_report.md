# TASK-006 변경 보고서: 계정 등록 신청 및 부서 필수 선택

## 작업 개요
- **작업 ID**: TASK-006
- **버전**: v1.5.0
- **작업일**: 2026-05-26
- **작업자**: AI

## 변경 사항

### 1. 계정 등록 신청 페이지 (`register.html`)
- 희망 아이디, 이름 입력
- 역할 선택 UI (카드형: 🎒학생, 👩‍🏫교사, 📋부서관리자)
- 소속 부서 필수 선택 (departments 테이블에서 로드)
- 신청 사유 (선택)
- 아이디 중복 확인 (기존 계정 + 대기중 신청)
- 신청 완료 시 안내 화면 (초기 비밀번호 1234 안내)

### 2. 로그인 페이지 연결 (`login.html`)
- "📝 계정 등록 신청" 버튼 추가
- 안내 문구: "계정이 없으신가요? 신청 후 관리자 승인을 받으면 이용 가능합니다"

### 3. 관리자 신청 내역 관리 (`admin/users.html`)
- 사용자 목록 상단에 "📝 가입 신청 내역" 섹션 추가 (대기 중인 신청만 표시)
- 승인 버튼: 자동으로 계정 생성 (초기 비밀번호 1234, 첫 로그인 시 변경 필수)
- 거부 버튼: 사유 입력 후 거부 처리
- 신청이 없으면 섹션 자동 숨김

### 4. 부서 필수 선택 조건
- 사용자 등록 시 소속 부서 필수 (`*` 표시)
- 부서 관리자 역할일 때 담당 관리 부서 필수 (`*필수` 표시)
- 유효성 검사: 부서 미선택 시 저장 불가

### 5. DB 변경
- `registration_requests` 테이블 신규 생성
  - `id`, `username`, `display_name`, `role`, `department_id`, `reason`
  - `status` (pending/approved/rejected)
  - `reviewed_by`, `review_note`, `reviewed_at`
- RLS 정책 4개 (INSERT/SELECT/UPDATE/DELETE)
