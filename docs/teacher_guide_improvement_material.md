# ⭐ 달란트 마을 교사 가이드 개선 자료

> 대상: 일반 교사 권한자(40+)  
> 활용: 15분 온보딩·시연 자료 보완용  
> 권장 기준: v3.26.1 / 2026-06-03 현재 사이트 기준

---

## 1. 핵심 메시지

**일반 교사는 담당 학생의 활동을 기록하고, 학생 명의로 상품을 대신 신청합니다.**  
화면이 다르게 보이면 오류보다 먼저 **권한, 담당 부서, 담당 반 설정**을 확인합니다.

유지해야 할 핵심 흐름은 다음과 같습니다.

- 출석은 빠르게, 활동 달란트는 항목 체크 후 지급 확정
- QR 수령은 학생에게 주는 기능이 아니라 교사 본인이 받는 기능
- 교사용 상품은 직접 신청, 학생용 상품은 대리 구매
- 대리 구매는 교사 달란트가 아니라 학생 달란트로 신청
- 대리 구매 주문은 교사 본인 `내 구매 상품`에 보이지 않을 수 있음

---

## 2. 가장 먼저 고칠 것

### 1) 버전 갱신

현재 표지의 `v3.23.0 기준`을 아래처럼 바꾸는 것을 권장합니다.

> 기준 버전: v3.26.1 / 2026-06-03 현재 사이트 기준  
> 대상: 일반 교사 40+ 권한자 온보딩 자료

### 2) 권한 배지 반복

일반 교사 전용 가이드라면 각 기능마다 권한 배지를 계속 보여주는 것이 좋습니다.

| 기능 | 권장 배지 |
|---|---|
| 내 달란트 | 로그인 사용자 |
| 달란트 수령 | 로그인 사용자 |
| 달란트 관리 | 일반 교사 40+ |
| 대리 구매 | 일반 교사 40+ |
| 사용자/상품/구매 관리 | 부서 담당 교사 60+ |
| 반환/보고서 | 부장 교사 80+ |

### 3) Q&A 정책 통일

현재 문서마다 Q&A 답변 권한 설명이 다르게 읽힐 수 있습니다. 실제 운영 정책을 확인해 아래 둘 중 하나로 통일해야 합니다.

- 일반 교사가 질문 확인·등록만 가능하다면: PDF와 교사 가이드 문구 유지, README 변경 이력 수정
- 일반 교사도 답변 가능하다면: PDF 13쪽과 사이트 교사 가이드 문구 수정

## 3. 일반 교사 기능 지도

| 기능 | 권한 | 교사에게 설명할 말 | 주의 문구 |
|---|---|---|---|
| 내 달란트 | 로그인 사용자 | 교사 본인의 잔액, 사용 대기, 구매 내역을 확인합니다. | 학생 주문은 교사 본인 화면에 보이지 않을 수 있습니다. |
| 달란트 수령 | 로그인 사용자 | 교사용 QR을 스캔해 교사 본인이 달란트를 받습니다. | 학생에게 달란트를 지급하는 화면이 아닙니다. |
| 달란트 관리 | 일반 교사 40+ | 담당 부서/반 학생에게 출석 또는 활동 달란트를 기록합니다. | 목록이 비면 담당 부서/반 설정을 확인합니다. |
| 대리 구매 | 일반 교사 40+ | 학생 명의로 학생의 달란트를 사용해 상품 구매 신청을 넣습니다. | 교사 달란트로 사는 것이 아닙니다. |
| 사용자/상품/구매 관리 | 부서 담당 교사 60+ | 담당 부서 범위 안에서 운영 관리 기능을 사용합니다. | 일반 교사 화면에는 보이지 않을 수 있습니다. |
| 반환/보고서 | 부장 교사 80+ | 운영 관리 기능으로 별도 권한이 필요합니다. | 일반 교사 교육에서는 참고 수준으로만 다룹니다. |

| 9쪽 달란트 관리 | “학생 목록이 안 보이면 담당 부서/반 설정부터 확인” 안내 박스 추가 |
| 11쪽 대리 구매 | 학생 명의 주문, 학생의 사용 가능 달란트 사용 문구를 크게 유지 |

| 13쪽 Q&A | 실제 Q&A 답변 권한 확인 후 PDF·사이트·README 문구 통일 |

## 5. 달란트 지급 시연 흐름

1. **학생 찾기**  
   담당 부서/반 학생 목록에서 대상을 선택합니다.

2. **출석 또는 항목 선택**  
   출석은 즉시 지급하고, 활동 달란트는 항목을 체크합니다.

3. **지급 확정**  
   중복 여부를 확인한 뒤 지급 확정 버튼을 누릅니다.

4. **상세 확인**  
   누가, 언제, 어떤 항목으로 지급했는지 확인합니다.

> 보완 문구: 일반 교사는 주로 `출석 버튼`과 `달란트 지급` 항목 체크를 사용합니다. 금액을 직접 입력하는 수동 적립/사용은 권한에 따라 보이지 않을 수 있습니다.

---

## 6. QR 수령 실패 대응 체크리스트

- 카메라가 열리지 않으면 브라우저 카메라 권한을 허용합니다.
- 위치 제한 QR은 브라우저 위치 권한과 실제 장소를 확인합니다.
- 학생용 QR인지 교사용 QR인지 대상 구분을 확인합니다.
- 유효기간이 지난 QR은 수령할 수 없습니다.
- 1회 수령 QR은 이미 받았다면 다시 받을 수 없습니다.

---

## 7. 대리 구매 실패 대응 체크리스트

- 담당 범위 밖 학생은 검색되지 않을 수 있습니다.
- 학생의 사용 가능 달란트가 부족하면 구매 신청이 제한됩니다.
- 품절 또는 비활성 상품은 신청 대상에서 제외될 수 있습니다.
- 학생 상품은 교사가 직접 구매하는 것이 아니라 대리 구매로 신청합니다.
- 대리 구매 주문은 학생 명의이므로 교사 본인 `내 구매 상품`에는 보이지 않을 수 있습니다.

---

## 8. 구매 상태 안내

| 단계 | 의미 | 안내 문구 |
|---|---|---|
| 구매 신청 | 신청만 완료 | 이 단계에서는 취소할 수 있습니다. |
| 상품 준비 | 담당자가 확인 | 상품을 준비하는 단계입니다. |
| 상품 구매 | 구매 확정 | 이때 달란트가 실제로 차감됩니다. |
| 상품 지급 | 전달 완료 | 상품을 받은 완료 상태입니다. |

> 취소 보완: 구매 신청 단계에서 취소하면 사용 대기 달란트가 복원되고, 주문은 구매 취소 상태로 처리됩니다.

## 9. 15분 시연 추천 순서

1. **로그인**: 교사 계정으로 접속하고 보이는 메뉴 확인
2. **달란트 관리**: 출석 버튼과 항목 체크 지급 시연
3. **내 달란트**: 교사 본인 잔액과 사용 대기 의미 설명
4. **QR 수령**: 교사용 QR 수령과 실패 체크리스트 안내
5. **상품 구매**: 교사용 직접 구매와 학생용 상품 조회 차이 설명
6. **대리 구매·Q&A**: 학생 선택 후 상품 신청, 문제 발생 시 Q&A 확인 위치 안내

## 10. 발표자가 그대로 읽어도 되는 마무리 멘트

> 오늘 기억할 것은 세 가지입니다. 첫째, 일반 교사는 담당 부서와 반 안에서 학생 달란트를 기록합니다. 둘째, 학생 상품은 교사 달란트가 아니라 학생 달란트로 대리 구매합니다. 셋째, 화면이 다르게 보이면 오류보다 먼저 권한과 담당 부서/반 설정을 확인합니다.
