CHO-Talents 페이지별 권한 설정 정리

작성 기준: v3.55.0 / 2026-06-29. 실제 소스의 js/codes.js, PERMISSION_RANK, initPage(), data-min-perm, role_page_access, js/nav.js 가이드 라우팅 기준으로 정리합니다.

1. 권한 등급 체계

0 비로그인 20 학생 40 일반 교사 60 부서 담당 교사 70 구매 담당 교사 80 부장 교사 90 전도사님 100 관리자 110 최고 관리자

최고 관리자는 permission_level='admin'이면서 is_super_admin=true인 계정이며, 클라이언트에서는 rank 110으로 계산합니다.

권한 라벨과 rank 메타는 v3.50.0부터 profiles.permission_level 코드 그룹 기준으로 관리합니다. 프론트엔드는 js/codes.js 기본 코드북을 사용하고, DB code_items.meta.rank가 있으면 해당 값을 우선합니다.

6. 24시간 세션 관리 및 리디렉트 로그

js/auth.js의 initPage()가 성공하면 startSessionTimer()가 자동 시작됩니다. 마지막 활동(클릭·키입력·스크롤·터치 등)으로부터 24시간 유휴 시 세션이 만료되고 로그인 페이지로 이동합니다. 활동 시간은 localStorage(cho_last_activity)로 멀티탭 간 공유되며, 탭 전환 시에도 만료 여부를 재검증합니다. v3.53.0부터 세션 없음/만료, 최초 로그인 이동, 권한 등급 부족, 허용 권한 불일치, role_page_access 차단은 AUTH_REDIRECT 로그에 요청 페이지, page_id, 필요/실제 권한, 이동 대상을 남깁니다.

7. 보조 권한 테이블

테이블용도적용 방식
role_page_access권한별 페이지 접근과 요소 숨김initPage() 통과 후 현재 page_id를 조회해 접근 보조 제한과 hidden_elements를 적용합니다.
role_page_features권한별 기능 설정값운영 설정 화면에서 관리합니다. 실제 실행 차단은 개별 화면의 rank 체크와 RLS/RPC가 우선입니다.
page_permissions레거시 권한 매트릭스admin/page-permissions.html에서 관리합니다. 신규 구현은 role_page_access를 우선합니다.
code_groups, code_items권한/사용자 유형 코드 마스터profiles.permission_level과 profiles.user_type의 라벨, 정렬, rank 메타를 관리합니다.
