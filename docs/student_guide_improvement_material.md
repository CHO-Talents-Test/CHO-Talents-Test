# ⭐ 달란트 마을 학생 가이드 개선안

> 학생이 혼자서도 길을 찾을 수 있도록, 기존 학생 발표자료와 사이트 안내 문서를 바탕으로 정리한 **꾸며진 수정·보완 제안서**입니다.

---

## 🎯 핵심 판단

| 구분 | 정리 |
|---|---|
| 현재 장점 | 발표 흐름이 쉽고, `달란트 모으기 → 내 달란트 확인 → 상품 신청 → Q&A` 구조가 명확함 |
| 주요 문제 | 실제 사이트, MD 문서, PDF 발표자료 사이의 **명칭·기능 설명·화면 기준**이 일부 다름 |
| 개선 방향 | 학생이 실제로 누르는 메뉴 중심으로 단순화하고, 막히는 상황에 대한 체크리스트를 추가 |

---

## 🧭 한눈에 보는 개선 포인트

| 개선 포인트 | 왜 필요한가요? | 반영 방식 |
|---|---|---|
| 🧭 메뉴명 통일 | `사용자 가이드`, `학생 가이드`가 섞이면 혼란스러움 | 학생용은 **학생 가이드**로 통일 |
| 🔐 첫 로그인 보강 | 승인 후 어떻게 로그인하는지 학생이 헷갈릴 수 있음 | 초기 비밀번호, 첫 로그인 변경, 비밀번호 약속 추가 |
| 📱 QR 실패 대처 | 현장에서 가장 자주 막히는 지점 | 카메라/기간/대상/이미 수령/위치 확인 체크리스트 추가 |
| 🏅 내 달란트 카드 설명 | PDF에는 카드 5개라 되어 있으나 설명 누락 가능 | `사용 완료 달란트` 설명 추가 |
| 🛒 구매 취소 안내 | 학생이 신청 후 달란트 변화를 오해할 수 있음 | 취소 시 사용 대기 달란트 복원 안내 |
| 🖼️ 학생 화면 캡처 | 교사/관리 메뉴가 보이면 학생이 따라 하기 어려움 | 실제 학생 계정 기준 캡처 사용 |

## ✅ 우선 수정 항목 TOP 8

### 1. 가이드 명칭 통일 `필수`
사이트와 문서에서 `사용자 가이드`로 남아 있는 표현을 학생 권한자 기준의 **학생 가이드**로 정리합니다.

### 2. 학생이 쓰는 메뉴만 먼저 설명 `필수`
학생용 자료에는 다음 메뉴를 우선 설명합니다.

- 달란트 적립
- 달란트 수령
- 내 달란트
- 상품 구매
- 내 구매 상품
- Q&A

### 3. 계정 신청·승인·첫 로그인 흐름 추가 `필수`
`계정 신청 → 승인 기다리기 → 초기 비밀번호로 로그인 → 새 비밀번호 변경` 흐름을 한 번에 설명합니다.

### 4. 내 달란트 카드 5개 설명 보완 `화면 일치`

| 카드 | 학생용 설명 |
|---|---|
| ⭐ 사용 가능 달란트 | 지금 상품 신청에 쓸 수 있는 달란트예요. |
| 🎁 상품 수령 예정 | 아직 받지 않은 상품 개수예요. |
| 🤝 사용 대기 달란트 | 신청했지만 아직 확정 전인 달란트예요. |
| ✅ 사용 완료 달란트 | 상품 구매로 실제 사용된 달란트예요. |
| 📈 누적 적립 달란트 | 지금까지 받은 달란트 총합이에요. |

### 5. QR 수령 실패 체크리스트 추가 `현장 대응`

- [ ] 카메라 사용을 허용했나요?
- [ ] 오늘 받을 수 있는 QR인가요?
- [ ] 이미 받은 QR은 아닌가요?
- [ ] 학생용 QR이 맞나요?
- [ ] 위치 확인이 필요한 QR인가요?

### 6. 구매 취소와 달란트 복원 설명 `불안 해소`
구매 신청 단계에서는 취소할 수 있고, 취소하면 사용 대기 달란트가 다시 돌아온다는 점을 넣습니다.

### 7. 학생 화면 기준 캡처로 교체 `시각 자료`
교사/관리/운영 메뉴가 섞인 화면은 학생에게 혼란을 줄 수 있으므로, 실제 학생 계정 기준 캡처로 교체합니다.

### 8. 안전 수칙 강화 `보안`
친구에게 비밀번호 알려주지 않기, 공용 기기 사용 후 로그아웃하기, 비밀번호 분실 시 선생님께 말하기를 넣습니다.

---

## 🔑 처음 시작하기 흐름

```text
계정 신청 → 선생님/관리자 승인 기다리기 → 처음 로그인 → 새 비밀번호 변경
```

### ✍️ 바로 넣을 문구

> 처음 로그인할 때는 선생님이 알려준 초기 비밀번호로 들어가요.  
> 처음 한 번은 새 비밀번호로 바꿔야 해요.  
> 비밀번호는 친구에게 알려주지 않고, 잊어버리면 선생님께 말해요.

---

## 📱 QR로 달란트 받기

### 성공 순서

1. `달란트 수령` 메뉴로 들어가요.
2. 카메라 버튼을 눌러요.
3. QR 코드를 화면에 비춰요.
4. 성공하면 내 달란트가 늘어나요.

### ✍️ 바로 넣을 문구

> QR이 안 되면 먼저 확인해요.  
> 카메라를 허용했나요? 오늘 받을 수 있는 QR인가요?  
> 이미 받은 QR은 아닌가요? 학생용 QR이 맞나요?  
> 위치 확인이 필요한 QR이면 위치 허용도 필요해요.

---

## 🛒 상품 신청과 구매 상태

```text
구매 신청 → 상품 준비 → 상품 구매 → 상품 지급
```

| 단계 | 학생용 설명 |
|---|---|
| 구매 신청 | 아직 선생님 확인 전이에요. 이때는 취소할 수 있어요. |
| 상품 준비 | 선생님이 상품을 준비 중이에요. |
| 상품 구매 | 구매가 확정되어 달란트가 실제로 사용됐어요. |
| 상품 지급 | 상품을 받은 완료 상태예요. |

### ✍️ 바로 넣을 문구

> 상품 구매 버튼은 바로 받는 버튼이 아니에요.  
> 먼저 구매 신청을 하고, 선생님이 확인한 뒤 준비와 지급이 진행돼요.  
> 구매 신청 단계에서는 취소할 수 있고, 취소하면 사용 대기 달란트가 돌아와요.

## 🧭 메뉴가 다르게 보일 때

> 필요한 메뉴가 안 보이면 선생님께 말해요.
