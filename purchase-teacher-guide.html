전체 부서의 상품 주문을 확인하고 구매 흐름을 진행하는 권한자용 안내입니다. 구매 담당 교사는 부서 담당 교사와 같은 관리 메뉴를 사용하면서, 구매 관리와 상품 배지에서 전체 부서 주문을 처리합니다.

🛒 구매 담당 교사의 핵심 업무

  1. 상품 > 구매 관리에서 전체 부서 주문이 들어왔는지 확인합니다.
  2. 칸반 카드에서 구매 신청 건을 눌러 처리할 주문만 필터링합니다.
  3. 실제 주문 확인 후 상품 준비 상태로 변경합니다. 이 전환만 Slack 상품 관리 채널 알림이 발송됩니다.
  4. 실제 구매가 끝났으면 상품 구매로 확정합니다. 이때 사용자의 달란트가 실제 차감됩니다.
  5. 상품 전달이 끝나면 상품 지급으로 완료 처리합니다.
구매 신청 단계에서는 달란트가 사용 대기로만 묶입니다. 실제 차감은 상품 구매 확정 단계에서 발생하므로, 실구매 전 확정하지 않도록 주의합니다.
구매 상태 이름, 색상, 표시 순서는 공통 코드 기준으로 관리됩니다. 구매 관리, 구매 통계, 내 구매 상품에서 같은 주문 상태는 같은 이름으로 표시됩니다.

🔔 구매 알림 기준

  • 학생/교사 구매 신청 또는 대리 구매 신청은 신청자 소속 부서 Slack 채널로 전송됩니다.
  • 구매 관리에서 구매 신청 → 상품 준비로 바뀔 때만 상품 관리 채널로 전송됩니다.
  • 상품 구매 확정, 상품 지급, 되돌리기, 취소는 현재 Slack 알림 대상이 아닙니다.
  • Slack 실패는 사용자 작업을 막지 않고 콘솔 경고만 남깁니다.

✅ 구매 전 확인 사항

상품 정보

상품명, 가격, 카테고리, 대상(학생용/교사용)이 실제 구매 품목과 맞는지 확인합니다. 카테고리와 대상은 공통 코드 기준 선택값으로 관리되며, 필요한 카테고리가 없으면 상품 등록/수정 모달에서 새 카테고리를 추가한 뒤 선택할 수 있습니다.

주문 대상

신청자 이름, 부서, 반, 대리 구매 여부를 확인합니다. 대리 구매는 학생 명의 주문입니다.

달란트 차감

상품 구매 확정 시점에 실제 차감됩니다. 잘못 확정했다면 권한자에게 반환/정정 절차를 요청합니다.

💰 달란트 상세 조회 (v3.48.0)

구매 확정 전후로 학생 달란트 잔액을 확인할 때 달란트 > 달란트 관리 > 상세 팝업을 활용합니다. 상세 이력은 페이지 이동과 페이지당 항목 수 선택을 지원하므로 오래된 적립/사용 내역도 같은 방식으로 확인합니다.

  • 상단 요약: 누적적립, 총사용, 총반환, 잔여 4칸 표시
  • 내역 유형: 적립 / 사용 / 반환 3종 구분
  • 총사용에는 실제 상품 구매 확정 건만 집계되며, 반환은 별도로 총반환에 표시됩니다.
