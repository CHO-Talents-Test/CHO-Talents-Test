👥 사용자 관리

  1. 관리 > 사용자 관리로 이동합니다.
  2. 상단 통계 카드(전체·관리자·부서 담당·교사·학생)를 클릭하면 해당 유형으로 목록을 필터링할 수 있으며, 선택된 카드는 강조 표시됩니다. 관리자는 admin·전도사·부장 교사, 부서 담당은 구매 담당·부서 담당 교사를 포함합니다.
  3. 사용자 목록은 교사/학생 그룹별로 분리되어 표시됩니다. 기본 정렬은 부서, 반, 이름 오름차순이며 각 그룹 PC 20명, 모바일 10명씩 페이징합니다.
  4. 가입 신청이 있으면 신청자의 이름, 부서, 신청 사유를 확인합니다.
  5. 사용자 유형과 권한을 선택한 뒤 승인하거나, 사유를 입력해 거부합니다.
  6. 각 사용자의 관리 ▾ 드롭다운에서 상세 정보, 수정, 부서 이동, 비밀번호 초기화, 삭제를 선택합니다.
  7. 상세 정보에서 아이디, 역할, 권한, 부서, 마지막 로그인, 신청 이력, 달란트 내역, 구매 내역을 확인할 수 있습니다.
부서 이동은 권한에 따라 즉시 처리되거나 승인 요청으로 생성됩니다. 담당 범위 밖의 사용자는 처리 권한이 없을 수 있습니다. 부서 담당 교사(60+)는 담당 부서 학생의 반 변경과 비밀번호 초기화가 가능하며, 삭제·부서 이동은 부장 교사(80+) 이상만 가능합니다.
최고관리자(Super Admin): is_super_admin이 TRUE인 계정은 수정 모달에서 역할, 권한, 소속 부서, 담당 부서 등 모든 필드를 제약 없이 변경할 수 있습니다.

📋 학생 일괄 등록

  1. 관리 > 학생 일괄 등록으로 이동합니다 (부장 교사 80+ 이상).
  2. 양식 다운로드 버튼으로 엑셀 파일(.xlsx)을 받습니다.
  3. 양식에 아이디, 이름, 부서, 반을 입력합니다.
  4. 작성한 엑셀 파일을 드래그하거나 클릭하여 업로드합니다.
  5. 미리보기에서 오류 항목(아이디 누락, 부서 없음 등)을 확인합니다.
  6. 일괄 등록 시작을 클릭하면 정상 데이터만 등록됩니다.
  7. 등록 결과에서 성공/실패 건수와 사유를 확인합니다.
초기 비밀번호는 "1234"입니다. 등록 후 학생에게 비밀번호 변경을 안내해주세요.

🏢 부서와 관리자 관리

부서 관리

  • 부서명을 등록하거나 설명, 반 개수를 수정합니다.
  • 반 개수가 0이면 반 구분 없이 운영합니다.
  • 부서 목록은 부서명 오름차순으로 표시되며, 우측 상단에서 페이지당 항목 수를 선택합니다.
  • 관리 드롭다운에서 소속보기, 수정, 삭제를 선택합니다.
  • 소속보기에서 관리자(100등급 이상)에게는 마지막 로그인 일시가 표시됩니다.

관리자 관리

  • 기존 교사 사용자를 검색해 운영 권한을 부여합니다.
  • 권한과 소속 부서, 담당 관리 부서를 지정합니다.
  • 구매 담당 교사는 역할명이 '구매 담당 교사'로 정확히 표시됩니다.
  • 관리자 목록은 권한 등급 내림차순, 부서, 이름 오름차순으로 표시됩니다.
  • 본인보다 높은 권한은 수정할 수 없습니다.

🎫 달란트 QR 관리

  1. 달란트 > 달란트 QR 관리로 이동합니다.
  2. 지급 대상이 학생인지 교사인지 선택합니다.
  3. 달란트 항목을 선택하거나 직접 설명과 금액을 입력합니다.
  4. 지급 제한 없음 또는 선착순 지급 인원을 선택합니다.
  5. 유효 기간을 지정일, 기간, 무기한 중에서 선택합니다. 지정일은 시작 시간과 종료 시간을 함께 설정할 수 있으며 해당 시간 안에서만 지급됩니다.
  6. 필요하면 위치 제한을 켜고 카카오맵 주소 검색으로 장소와 반경을 지정합니다. 반경은 100m, 200m, 300m, 400m, 500m, 1km~5km 중 선택하며 기본값은 500m입니다. 수령자 기기에서 위치 권한이 차단되면 수령 화면에 알림창과 안내 메시지가 표시됩니다.
  7. QR 코드를 생성한 뒤 인쇄하거나, QR 목록과 수령자 목록을 확인합니다. 두 목록은 페이징과 페이지당 항목 수 콤보 박스(3~30개, 계정별 DB 저장)를 지원합니다. 반환된 달란트는 수령자 이름 옆에 반환 배지가 표시되며, 목록 상단에는 총 N명 수령 (반환 M명) 형식으로 집계됩니다.
QR을 수정하면 기존 코드 값을 그대로 고치는 것이 아니라 새 QR 코드가 만들어집니다. 이미 배포한 QR을 바꿀 때는 새 코드로 다시 안내하세요.

🛒 상품과 구매 관리

상품 관리

  • 교사용/학생용 그룹별로 상품이 분리 표시됩니다(모바일 10개/PC 20개 페이징). 테이블 헤더를 클릭하면 카테고리, 상품명, 가격 기준으로 정렬할 수 있습니다.
  • 상품명, 코드 마스터 기준 카테고리, 가격, 설명, 이미지 또는 이모지를 입력합니다. 필요한 카테고리가 없으면 상품 등록/수정 모달의 + 추가 버튼으로 새 카테고리명과 이모지를 등록한 뒤 바로 선택할 수 있습니다.
  • 관리 드롭다운에서 수정 또는 삭제를 선택합니다.

구매 관리

  • 칸반보드 형태의 상태별 카드에 개수가 실시간 표시되며, 클릭 시 해당 상태로 필터됩니다.
  • 상태명, 색상, 정렬 순서는 product_orders.status 코드 기준으로 상점/내 구매 상품/구매 통계와 동일하게 표시됩니다.
  • 부서와 기간 필터로 필요한 주문만 조회합니다.
  • 구매 담당 교사는 전체 부서 주문을 조회하고 처리하며, 상품 메뉴 배지도 전체 범위로 집계됩니다.
  • 관리 드롭다운에서 상태 변경이나 되돌리기를 선택합니다.
  • 구매 확정 단계에서 달란트가 실제 차감됩니다.

📢 공지 관리

운영 > 공지 관리는 전도사님(90+) 이상 권한에서 사용할 수 있습니다. 등록된 공지는 목록에서 과거 내용까지 확인할 수 있고, 각 공지의 공지 토글로 활성/비활성 상태를 바꿉니다.

1 공지 등록 버튼을 눌러 제목과 내용을 입력합니다.
2 활성 공지로 표시할지 선택하고 저장합니다.
3 목록의 보기로 기존 내용을 확인하고 수정으로 다시 저장합니다.
4 활성 공지는 로그인 사용자의 메인 화면에 팝업으로 표시됩니다.

사용자가 팝업 하단의 다시 열지 않음을 선택하면 해당 계정에는 그 공지가 다시 표시되지 않습니다. 공지 등록, 수정, 활성 토글은 로그와 작업 이력에 남습니다.

🔐 관리자 전용 설정

📊 활동 로그 (한글화)

모든 활동 로그의 액션 이름이 한글로 표시됩니다.

1 운영 → 로그에서 활동 로그를 확인합니다.
2 액션 열에 한글 라벨이 표시되며, 영문 키는 작은 글씨로 병기됩니다.
3 상세 내역의 키도 한글로 표시됩니다 (대상, 변경내역, 오류 등). 적재 실패는 브라우저 콘솔 오류로 확인할 수 있습니다.

미확인 ERROR 이상 로그는 확인 완료 처리를 남겨야 합니다. 삭제는 삭제 대기 방식으로 처리됩니다. v3.40.0부터 페이지 조회(PAGE_VIEW) 로그는 기록되지 않습니다. 액션 라벨은 코드 마스터와 details._actionLabel을 기준으로 표시됩니다. 보호 페이지가 로그인 화면이나 메인 화면으로 이동한 원인은 AUTH_SESSION_MISSING, AUTH_PROFILE_LOAD_FAIL, AUTH_REDIRECT, AUTH_PAGE_ACCESS_CHECK_FAIL 로그에서 확인합니다.

24시간 유휴 자동 로그아웃

로그인 후 24시간 동안 사이트를 사용하지 않으면 세션이 만료되어 자동 로그아웃됩니다. 페이지 조회, 클릭, 스크롤, 키 입력 등 활동이 있으면 24시간 타이머가 처음부터 다시 시작됩니다. 여러 탭을 열어 두어도 활동 시간은 공유됩니다. 만료로 이동한 경우 AUTH_REDIRECT 로그에 만료 기준이 함께 기록됩니다.

✅ 운영할 때 기억할 점

  • 비밀번호는 확인할 수 없고, 필요한 경우 1234로 초기화만 할 수 있습니다.
  • 구매 신청은 바로 차감이 아니며, 구매 확정 시점에 달란트가 차감됩니다.
  • 달란트 반환은 거래 ID 기준으로 1회만 가능하며, 반환된 항목은 지급 모달의 취소 상태와 동기화됩니다. 반환 건은 총반환에 집계되고 총사용에는 포함되지 않습니다. 총사용에는 실제 상품 구매 확정 건만 집계됩니다.
  • 권한이 낮은 계정은 일부 버튼이 보이지 않거나, 보이더라도 처리 권한이 제한될 수 있습니다.
  • 삭제 대기 처리된 로그나 상품은 일반 목록에서 숨겨지지만 기록은 남습니다.
  • 작업 이력은 별도 저장소가 아니라 활동 로그 중 관리 작업만 필터링해서 보여줍니다.
  • 상품 카테고리는 상품 등록/수정 모달에서 추가할 수 있습니다. 새 권한/상태/로그 유형을 추가할 때는 js/codes.js 기본값과 DB code_items 마스터를 함께 확인합니다.
  • 공지 관리는 전도사님(90+) 이상이 담당합니다. 활성 공지는 모든 로그인 사용자에게 팝업으로 보이며, 사용자가 다시 열지 않음을 선택하면 계정별 숨김 기록이 남습니다.
  • QR 위치 제한을 쓰려면 사용자가 수령할 때 브라우저 위치 권한을 허용해야 합니다. 차단된 경우 수령 화면에서 알림창이 표시되고 QR_LOCATION_PERMISSION_BLOCKED 로그가 기록됩니다. 위치 제한 반경의 기본값은 500m입니다.
  • 목록형 화면과 달란트 관리 상세 이력은 공통 페이징 규칙을 사용합니다. 현재 페이지 주변과 첫/마지막 페이지를 표시하고, 페이지가 7개를 초과하면 말줄임표(...)로 축약됩니다. 각 그리드/모달의 페이지당 항목 수 콤보 박스(3~30개)로 개인 설정이 가능하며 DB에 저장됩니다.
  • 주요 관리 그리드는 기본 정렬을 갖습니다. 사용자/달란트 관리는 부서·반·이름, 관리자 관리는 권한 등급·부서·이름, 부서 관리는 부서명, 통계 화면은 각 통계 목적에 맞는 합계·개수·부서·이름 순서로 정렬됩니다.
